## Option 1: Technical deep-dive (best for dev audience)

**Title:** I built an open-source tool that keeps your AI coding agent's context from going stale.

**Body:**

Your AI coding agent is only as good as the context it receives. But maintaining CLAUDE.md, AGENTS.md, and .cursor/rules files is a manual chore that most teams neglect — until their agent starts hallucinating because it's working with outdated information.

I ran into this problem constantly. My CLAUDE.md would start accurate, then slowly drift as the codebase evolved. New directories, changed dependencies, different commands — but the context file stayed frozen.

So I built RepoMemory — an open-source tool that:

🔍 Scans any repository and detects language, framework, structure, and existing AI context files
📊 Audits context quality across 7 dimensions (Architecture, Commands, Conventions, Off-limits, Testing, Deployment, Freshness)
🎯 Gives a 0-100 score with specific improvement suggestions per dimension
📝 Generates missing context files for any AI coding agent
👁️ Detects drift — git-aware analysis with risk scoring, alerts when code changes but context doesn't

It ships as:
- Web app (Next.js 15, dashboard with timeline and comparison)
- CLI (npx repomemory scan .)
- MCP server (8 tools for AI agents)
- VS Code extension
- GitHub Action (auto-check PRs)

Everything is 100% local-first. Your code never leaves your machine.

Stack: Next.js 15, TypeScript, shadcn/ui, ~15 core modules, 24 tests, 16 routes.

MIT licensed. PRs welcome.

🔗 https://github.com/PraveenJayaprakash-JP/repomemory
🌐 Live demo: https://repomemory-omega.vercel.app
